Big Data · definition

The 4 V's of Big Data.

Big Data không chỉ là "nhiều data". CFA định nghĩa qua 4 chiều: Volume, Velocity, Variety, Veracity — vượt qua capability của traditional databases.

V1
Volume
Quy mô dataset — terabytes, petabytes. Ví dụ: tick-by-tick order book, satellite imagery.
V2
Velocity
Tốc độ data arrive & process. Real-time streaming: market ticks, Twitter feed, IoT sensors.
V3
Variety
Nhiều loại: structured (CSV), semi-structured (JSON, XML), unstructured (text, image, audio).
V4
Veracity
Độ tin cậy & chất lượng — missing values, noise, bias. "Garbage in, garbage out".

Pre-processing

Data preparation.

80% thời gian ML project là data prep. 4 bước cốt lõi cho CFA L1:

  1. Cleaning — sửa lỗi, xử lý missing Remove duplicates, fix typos, handle NaN (drop hoặc impute bằng mean/median/forward-fill). Identify outliers.
  2. Wrangling (munging) — reshape & merge Pivot wide/long, join datasets, parse dates, extract features. Biến raw thành analyzable form.
  3. Aggregation — roll-up Tick data → minute bars → daily returns. Summary statistics theo group/time bucket.
  4. Normalization / standardization — đưa về same scale Min-max [0,1] hoặc z-score (x−μ)/σ. Cần cho ML distance-based (k-means, SVM) để feature không dominate nhau.

End-to-end workflow.

Machine Learning · overview

The ML taxonomy.

CFA L1 phân ML thành 4 loại theo cách learning. Phân biệt dựa trên có labels hay không và cách feedback.

01
Supervised
Có labels (Y known). Learn mapping X → Y. Regression (continuous Y), classification (categorical Y).
02
Unsupervised
Không labels. Tìm pattern/structure trong X. Clustering, dimension reduction.
03
Deep Learning
Neural networks nhiều hidden layers. Có thể supervised hoặc unsupervised. Đặc biệt mạnh cho image, text, audio.
04
Reinforcement
Agent tương tác với environment, learn qua reward/penalty. Trial-and-error. Ví dụ: AlphaGo, trading bot.

Supervised learning

Learning with labels.

§ Core idea Dataset có cả feature X và target Y. Algorithm học mapping f: X → Y từ training data, rồi predict Y cho new X.
Regression — predict continuous Y
Output
Số thực (price, return, revenue)
Algorithms
Linear / penalized (LASSO, Ridge), tree-based, neural nets
Example
Predict next-day stock return từ macro features
Classification — predict category
Output
Class label (up/down, default/safe, spam/ham)
Algorithms
Logistic regression, SVM, decision trees, random forest, neural networks
Example
Credit scoring: default vs non-default

§ SVM tìm hyperplane tách 2 class với maximum margin. Decision tree split theo feature tạo if-else rules. Random forest = ensemble của nhiều decision trees (bagging) → giảm variance, chống overfit.

Unsupervised learning

Finding structure without labels.

  1. Clustering — nhóm observations similar lại k-means: chia data thành k clusters minimize within-cluster variance. Ứng dụng: customer segmentation, stock grouping theo behavior, regime detection.
  2. Dimension reduction — giảm số features PCA (Principal Component Analysis): project data lên axes capture most variance. Ứng dụng: từ 500 stocks → 3-5 factor components; visualization high-dim data.
  3. Anomaly detection — tìm outlier Observation không fit với cluster nào → fraud, market manipulation, trading error.
§ Key distinction Supervised needs labeled training data (Y known). Unsupervised không cần labels — algorithm tự tìm structure. Nếu đề cho dataset "không có target variable" → unsupervised.

Model generalization

Over- vs underfitting.

Trade-off trung tâm của ML: bias-variance tradeoff. Model phải đủ flexible để fit data nhưng đủ simple để generalize.

Underfitting (high bias)
Behavior
Model quá đơn giản — miss patterns
Symptom
High error trên cả training và test set
Fix
Model phức tạp hơn, thêm features, giảm regularization
Overfitting (high variance)
Behavior
Model học cả noise — memorize training data
Symptom
Low training error NHƯNG high test error (gap lớn)
Fix
Cross-validation, regularization (L1/L2), thêm training data, giảm complexity

§ Dấu hiệu overfit trong đề CFA: "model performs excellent on training data but poorly on new/out-of-sample data". Đó là overfit — đáp án là regularization hoặc cross-validation.

Dataset splitting

Training · validation · test.

§ Why 3 sets? Tách data thành 3 phần để honestly evaluate model. Common split: 60% / 20% / 20% hoặc 70% / 15% / 15%.
  1. Training set (~60%) — fit parameters Algorithm học pattern ở đây. Weights, coefficients được optimize trên training data.
  2. Validation set (~20%) — tune hyperparameters Thử nhiều configurations (learning rate, tree depth, regularization strength). Chọn version performs tốt nhất trên validation.
  3. Test set (~20%) — final unbiased estimate CHỈ touch ONE time ở cuối. Report performance trên test như true out-of-sample estimate. Touch nhiều lần → data leakage → overfit test set.

§ K-fold cross-validation: chia training thành K folds, train K lần — mỗi lần giữ 1 fold làm validation. Trung bình performance → robust hyperparameter selection khi data ít.

NLP biến unstructured text (news, filings, tweets, earnings transcripts) thành numeric features cho ML models. Alpha source mới của hedge funds.

02
Sentiment analysis
Classify text: positive, negative, neutral. Ví dụ: Bloomberg news → sentiment score → trade signal.

Topic modeling
Unsupervised (LDA): tự discover topics từ corpus. Tag filings theo theme để track exposure.
04
Named entity recognition
Trích xuất: tên công ty, ticker, person, date, dollar amount — feed vào downstream systems.

Risks & caveats

Không phải magic.

CFA code of ethics yêu cầu analyst hiểu limitations của tool mình dùng. Big data/ML có 4 nhóm risk quan trọng:

  1. Data quality — bias, missing, survivorship Training data có sample selection bias → model inherit bias đó. Historical data có thể không đại diện cho future regime.
  2. Model risk — black box, overfitting Deep learning khó explain. Model fit quá khớp historical → backtest đẹp, live performance tệ. Cần model validation, stress test.
  3. Regulatory compliance — GDPR, data privacy Thu thập PII phải tuân thủ GDPR (EU), CCPA (California). Alternative data source phải có legal sourcing — không được insider info.
  4. Ethics — fairness, transparency Credit model discriminate theo race/gender (proxy variables) vi phạm pháp luật. Fiduciary duty → explain decisions đến clients.

Lesson takeaway

The big picture.

Core mental model
Big Data = 4 V's. Raw data qua pipeline (collect → clean → prepare → model → deploy → monitor) → ML. ML có labels → supervised. Không có labels → unsupervised. Feedback qua reward → reinforcement. Nhiều layers → deep. Model phải balance bias-variance, dùng train/val/test + cross-validation + regularization để tránh overfit.

Identify ML approach.

§ Vignette
A portfolio manager wants to group 500 stocks into clusters based on similarity of their return patterns over the past five years. She has no pre-defined categories and wants the algorithm to discover natural groupings. Which machine learning approach is most appropriate?
  1. ASupervised regression — predict future returns from features.
  2. BSupervised classification — assign each stock to a sector label.
  3. CUnsupervised clustering (k-means) — discover groupings without labels.
CFA Level I QUANT · 02.11 — Q1 ORIGINAL
Quant · 02.11 18 / 28
Hint · Question 01

Spot the keywords.

§ Keywords Group stocks into clusters. No pre-defined categories. Discover natural groupings từ data.
  1. Hint 1 "No pre-defined categories" = KHÔNG có labels Y. Supervised yêu cầu labels → loại A và B.
  2. Hint 2 "Group... based on similarity... discover natural groupings" = definition của clustering — unsupervised task tìm structure trong X.
  3. Hint 3 k-means là clustering algorithm điển hình. Chia N observations thành K clusters minimize within-cluster variance.

Dịch đề.

§ Vignette
Một portfolio manager muốn nhóm 500 cổ phiếu thành các clusters dựa trên sự tương đồng của return patterns trong 5 năm qua. Cô ấy không có các categories định sẵn và muốn algorithm tự discover natural groupings. ML approach nào phù hợp nhất?
  1. ASupervised regression — predict future returns từ features.
  2. BSupervised classification — gán mỗi stock một sector label.
  3. CUnsupervised clustering (k-means) — tự discover groupings, không cần labels.
CFA Level I QUANT · 02.11 — Q1 VIỆT
Quant · 02.11 20 / 28
Answer · Question 01

Answer: C — Unsupervised clustering.

✓ Correct — C
Không có labels (no pre-defined categories) + task là discover groupings → unsupervised clustering. k-means chia N stocks thành K clusters theo similarity của return pattern — đúng scenario.
✗ Wrong — A
Regression cần target Y liên tục (future return). Đề không yêu cầu predict value, chỉ yêu cầu group. Supervised regression cần labels training — đề nói KHÔNG có.
✗ Wrong — B
Classification cần labeled training set (stock → sector). Đề nói "no pre-defined categories" — nghĩa là không có labels để train. Nếu đã có sector labels thì không cần ML để gán.
Key takeaway
Decision rule: có labels + predict Y → supervised. Không có labels + tìm structure → unsupervised. Keywords "discover", "group", "segment", "no labels" → unsupervised. Keywords "predict", "classify" + có training labels → supervised.

Dịch đề.

§ Vignette
Một quant analyst dùng 3 data sources cho trading model: (1) OHLCV giá cổ phiếu hàng ngày trong SQL database, (2) ảnh vệ tinh của bãi đỗ xe trung tâm thương mại, (3) earnings call transcripts (text). Phát biểu nào về data types chính xác nhất?
  1. ACả 3 đều structured vì đến từ nguồn xác định.
  2. BOHLCV là structured; satellite images và transcripts là unstructured.
  3. COHLCV và transcripts là structured; chỉ satellite images là unstructured.
CFA Level I QUANT · 02.11 — Q2 VIỆT
Quant · 02.11 24 / 28
Answer · Question 02

Answer: B.

✓ Correct — B
OHLCV trong SQL = rows (date) × columns (O,H,L,C,V) → structured. Satellite image là pixel matrix, không fit schema relational → unstructured. Earnings call transcript là free-form text, không có column schema → unstructured.
✗ Wrong — A
Định nghĩa structured dựa trên form (có fit vào table hay không), KHÔNG dựa trên source reliability. Data từ nguồn đáng tin vẫn có thể là unstructured (ví dụ: Bloomberg news).
✗ Wrong — C
Text transcript là unstructured, không phải structured. Mặc dù có thể store trong database column, content bên trong là free text chưa có schema — phải NLP mới extract được features.
Key takeaway
Structured = fit tự nhiên vào rows & columns (SQL table). Semi-structured = có tags/hierarchy nhưng flexible schema (JSON, XML). Unstructured = text, image, audio, video — cần transform (NLP, CV) trước khi ML consume. 80-90% data doanh nghiệp ngày nay là unstructured.

Diagnose the problem.

§ Vignette
An analyst trains a deep neural network to predict stock returns. The model achieves 98% accuracy on the training set but only 52% accuracy on the out-of-sample test set. Which issue is the model most likely suffering from, and what is the best remedy?
  1. AUnderfitting — add more layers and features to the model.
  2. BOverfitting — apply regularization and cross-validation to improve generalization.
  3. CData leakage — the training and test sets are identical, so retrain from scratch.
CFA Level I QUANT · 02.11 — Q3 ORIGINAL
Quant · 02.11 26 / 28
Hint · Question 03

Spot the keywords.

§ Keywords Training accuracy 98%, test accuracy 52%. Gap 46 percentage points. Deep neural network (high capacity).
  1. Hint 1 Dấu hiệu điển hình: training error thấp, test error cao → overfitting. Model memorize training data nhưng không generalize.
  2. Hint 2 Deep NN có nhiều parameters → high capacity → prone to overfit, đặc biệt khi training data ít. Cần regularization (L1/L2, dropout) để constrain model.
  3. Hint 3 Underfitting có cả training và test đều kém (ví dụ 60%/55%). 98%/52% KHÔNG phải underfit. Data leakage sẽ cho test accuracy cao bất thường — opposite của đây.

Practice · Question 03 · Tiếng Việt

Dịch đề.

§ Vignette
Một analyst train deep neural network để predict stock returns. Model đạt 98% accuracy trên training set nhưng chỉ 52% accuracy trên out-of-sample test set. Model most likely đang gặp vấn đề gì, và remedy tốt nhất là gì?
  1. AUnderfitting — thêm layers và features cho model.
  2. BOverfitting — áp dụng regularization và cross-validation để cải thiện generalization.
  3. CData leakage — training và test set trùng nhau, phải retrain từ đầu.
CFA Level I QUANT · 02.11 — Q3 VIỆT
Quant · 02.11 28 / 28
Answer · Question 03

Answer: B — Overfitting.

✓ Correct — B
Signature pattern: high training accuracy (98%) + low test accuracy (52%) = overfitting. Model đã memorize noise của training data. Remedies: (1) Regularization (L1/L2, dropout) — penalize complex weights, (2) Cross-validation — validate trên multiple folds để catch overfit sớm, (3) thêm training data, (4) giảm model complexity.
✗ Wrong — A
Underfitting = cả training và test đều kém (high bias). Training 98% là excellent — model không underfit. Thêm layers/features làm overfit tệ hơn.
✗ Wrong — C
Data leakage (train/test overlap) sẽ cho test accuracy cao bất thường, không phải 52%. Ở đây test accuracy THẤP hơn training nhiều — opposite của data leakage.
Key takeaway
Decision rule cho exam: training ↑ test ↓ với gap lớn → overfit → fix bằng regularization + CV. Training ↓ test ↓ cả 2 đều kém → underfit → tăng complexity. Deep NN đặc biệt prone to overfit khi data ít.
